Webauthn - Proof of Concept

Problem statement:

There are several customers who login to Net Banking using the same device most of the time. Still they need to complete two factor authentication to do the transactions as it is mandated by RBI. Identify alternate modes of authentication which can ensure safety and convenience to customers.

Solution:

I created a webauthn based proof of concept, which enables users to login to Net Banking by using inbuilt biometrics and security keys. User can use any of the biometrics like faceID, Windows Hello, PIN and fingerprint which are available on the device. Users can also use security keys provided by different third party manufacturers. This reduces risk of phishing attacks as there are no passwords involved and security keys are only accessible by Net Banking website.

Dynamic QR - Proof of Concept

Problem statement:

Allow users to login to a website using mobile banking app or facilitate the login of one device through another. Ensure that both devices are in same location.

Solution:

I created a prototype allows login to a new device using an existing device using QR code. This QR code is only valid for few seconds and changes in set frequency. This ensures that the fraudsters cannot login even if they send this QR to users and make them scan as QR expires by the time users scan it in most of the cases.

Proximity - Proof of Concept

Problem statement:

Users can login to their account on a new device using a existing device where user is already logged in. Users need to scan a QR code shown on old device using New device to activate. But fraudsters posing as bank employees started exploiting this feature by asking users to send the image of QR via mail or whatsapp. To protect users from vishing attacks identify a better authentication method.

Solution:

By drawing inspiration from google pay and amazon pay apps, which using ultra low frequency sound to transmit transaction related data, I developed a prototype to use sound to authenticate. The prototype was developed using the APIs provided by google as a part of google play services. I later used SDKs provided by a developer named chirp which worked better tha google APIs. This prototype sends security key as a lowe frequency sound which can only be heard by another device. It is difficult to record and replicate sound unlike image in QR. This also ensures that the devices are in proximity and reduce the risk.

TOTP - Proof of Concept

Problem statement:

users of the Net Banking were facing issue in receiving OTPs through SMS as some of them live in areas with weak or no connectivity. There several corporate customers who live in tch parks which have connectivity issues because of lower bandwidths. To reduce dependency on service providers for delivering SMS and reducing costs to the bank alternates were explored

Solution:

I created a prototypes which can provide Time based or count based OTPs to the customer. Any user with Google authenticator or similar authenticator apps can use this functionality. By doing analysis of the pros and cons of the two types of open source offline OTPs, I concluded that Time based OTP is the better of two. Vendors are being explored to facilitate this solution as per guidelines given by RBI.
